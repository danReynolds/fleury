## Reversals

- 2026-06-01: Reversed "Keep command-palette result rows eagerly mounted for
  semantic testing." `SB.8` showed that eager row mounting kept correctness but
  made 1000-command palette filtering and settle latency too slow for launch
  claims. The replacement policy is lazy visible-row mounting with aggregate
  collection semantics, stable command-id search, and tests that render before
  asserting row-level semantics.
- 2026-06-01: Refined "Build `SearchPanel` as a general result surface, not a
  command-palette variant or ranking engine." The demo app and command
  palette showed that small/medium result surfaces should have first-party
  ranked matching. The replacement boundary is: `SearchResultIndex` owns
  sanitized result ranking, while apps still own corpus construction,
  debouncing, remote calls, worker/off-main execution, and paging policy.
